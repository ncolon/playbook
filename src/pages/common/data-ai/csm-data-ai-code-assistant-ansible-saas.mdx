## Overview
As a CSM, your focus is to guide and nurture your customers who have a watsonx Code Assistant for Red Hat Ansible Lightspeed entitlement. Your job is to guide the customer through provisioning to onboarding, helping with their first use case, and continuing to expand their use cases.

You can learn more about watsonx Code Assistant for Red Hat Ansible Lightspeed on the <a href='https://pages.github.ibm.com/csm-playbook/playbook/common/acceleration-plays/watsonx-code-assistant-ansible-play/' target='_blank' rel='noreferrer noopener'>Acceleration Play page</a> and  <a href='https://ibm.seismic.com/Link/Content/DCTbBjRdfjJXpGTCV4j3gT4BC8Wj' target='_blank' rel='noreferrer noopener'>Prospecting Product Playbook</a>.

| IBM CSM | watsonx Code Assistant for Ansible SaaS instance provisioned and the customer has access. |**Technology Expert Labs Offerings:** <a href="https://ibm.seismic.com/Link/Content/DCdJJgXcXcjQMGFF6g8X46hhW4X3" target='_blank' rel='noreferrer noopener'>**Custom Service:**</a><br/>  - Custom SOW <br/><br/> **Sizing and planning** <br/> watsonx Code Assistant for Ansible doesn't provide different sizing and topology options during provisioning. <br/><br/> **Provisioning the service** <br/>Use the <a href='https://cloud.ibm.com/docs/watsonx-code-assistant?topic=watsonx-code-assistant-getting-started' target='_blank' rel='noreferrer noopener'>Getting Started section</a> of the watsonx Code Assistant for Ansible Documentation along with the <a href='https://ibm.ent.box.com/s/vt5bhdeh8pq65j20wlez3f0b5ias0m0p' target='_blank' rel='noreferrer noopener'>CS IBM Cloud onboarding guide</a> to guide your customer through the process of provisioning the watsonx Code Assistant for Ansible service. |
